## 1. Summary

`std.numerics` is the library home for working with Mycelium's honest bounds: it lets a caller *construct* a bound (ε or δ) with its basis, *attach* it to a value (the `Approx<T>` carrier), *compose* bounds through the M-201/M-202 kernels, *inspect / EXPLAIN* a bound, and *check* a claimed certificate through the M-203 tier-i checker. Its **honesty crux** is C2 in its most literal form (the whole point of the module): **a helper's guarantee tag is exactly what its basis supports and is never upgraded** — strength composes by **meet** (weakest-wins), an op that lacks a sound propagation rule **refuses** rather than fabricates a bound, and the tier-i checker **re-validates** every composed certificate (a claim tighter than the re-derivation is `Rejected`, never silently passed). It is a Ring-1 capability surface (RFC-0016 §4.2): it *consumes* the landed `mycelium-numerics` kernels + the shared certificate and **adds no trusted code** (KC-3) — it is a certificate/EXPLAIN **consumer**, not a producer of trusted bound algebra.

## 2. Scope & module boundary

- **In scope:** the helper surface over the ADR-010 kernels —
  - the **`Approx<T>` carrier**: a value paired with its `{Bound, strength}` (the carrier-shape decision math.md §7 Q1 and dense.md §7 Q5 route here — homed in §3 / §7 Q1 below);
  - **bound constructors**: build an `ErrorBound{eps, norm, basis}` / `ProbabilityBound{delta, basis}` (ADR-011: every bound carries its basis) — including the explicit, always-flagged `declared(...)` constructor (`UserDeclared` basis);
  - **composition helpers**: thread two `Approx`/`Bound`s through the M-201 ε monoid (`compose_error_bound`) or the M-202 δ monoid (`union`), `strength` by `meet`, basis re-derived to match (Proven⊕Proven stays Proven under the affine-composition citation; …⊕Declared→Declared);
  - the **one sanctioned cross-kernel rule** `accuracy_to_probability` (the ε→δ coupling; ADR-010 §4) surfaced as an explicit, EXPLAIN-able helper — no other cross-kernel mixing;
  - **inspection / EXPLAIN**: project a bound's `{eps|delta, norm, basis, strength}` and *why* it holds (which theorem / which empirical fit / declared-and-unverified);
  - **checking**: re-validate a claimed certificate via the M-203 tier-i checker (`Valid`/`Rejected`/`Malformed`), surfaced as a `Result` whose error is a structured refusal record (RFC-0013).
- **Out of scope (and who owns it):**
  - the **trusted ε/δ kernels + the certificate record + the tier-i checker algorithm** → `mycelium-numerics` (M-201/202/203, ADR-010), the small trusted base. This module *calls* them; it does not re-implement the affine arithmetic, the union bound, or the checker (KC-3).
  - **numeric functions** (`sqrt`/`exp`/`log`/`pow`/trig) → `std.math` (M-525). `math` *consumes* this module's carrier + composition; it does not define the bound machinery.
  - **numeric-type conversion** (lossy/lossless scalar cast) → `std.cmp`/`convert` (M-532).
  - **representation change** (`F32→BF16`, `Dense↔VSA`) → `std.swap` (M-516) — certificate-carrying.
  - **VSA crosstalk / capacity bound *content*** → `std.vsa` (M-513) / RFC-0003 (the concentration-inequality theorems); ADR-010 §5 is explicit that the verified-numerics machinery is the *pattern and home* for ε/δ but does **not** supply VSA crosstalk content. This module surfaces the `CrosstalkBound`/`CapacityBound` *carriers* (bound.schema.json) but never fabricates their magnitudes.
- **Ring & layering:** Ring 1, Tier A (RFC-0016 §4.2/§4.3). It **wraps** the landed `mycelium-numerics` kernels + shared certificate as an ergonomic helper library; it defines **no new bound kind** (the four are closed in `bound.schema.json`) and **enlarges no trusted base** (KC-3). It is a certificate/EXPLAIN consumer, not a producer of trusted bounds — the checker, not this layer, is the trust anchor.

## 3. Exported-op surface (design sketch)

A design sketch — enough to fix the surface and feed the guarantee matrix, not a committed grammar. Value-semantic, immutable-by-default. Fallible ops return `Result`; an op that lacks a sound bound rule **refuses** (explicit `Err`), never fabricates a bound (C1). This module **homes the `Approx<T>` carrier** that `math` (§7 Q1) and `dense` (§7 Q5) deferred: an `Approx<T>` is a plain value plus its `Meta`-attached `{Bound, strength}` (RFC-0001 §4.3) — **not** a parallel numeric type, so it never forks the value model (the resolution is argued in §7 Q1).

```
// illustrative signatures (not a committed surface)

// --- the carrier: value + its honest {Bound, strength} (RFC-0001 §4.3 Meta projection) ---
type Approx<T>            // = a Value<T> whose Meta carries { bound: Bound, guarantee: Strength }
                          //   an EXACT value carries NO bound (M-I1) — Approx<T> with bound=None ≡ exact T

fn exact<T>(x: T) -> Approx<T>                       // total; tag Exact, bound = None (M-I1)
fn value_of<T>(a: Approx<T>) -> T                    // total; project the carried value
fn bound_of<T>(a: Approx<T>) -> Option<Bound>        // total; None iff Exact (M-I1)
fn strength_of<T>(a: Approx<T>) -> Strength          // total; the lattice tag

// --- bound constructors (ADR-011: every Bound carries its basis) ---
fn error_bound(eps: Real, norm: NormKind, basis: BoundBasis) -> Result<Bound, NumErr>
    // Err(BadEps) if eps < 0 (range-checked, mirrors the M-203 constructor)
fn prob_bound(delta: Real, basis: BoundBasis) -> Result<Bound, NumErr>
    // Err(BadDelta) if delta not in [0,1] (out-of-range δ refused — M-203 constructor)
fn declared<T>(x: T, b: Bound) -> Approx<T>          // basis = UserDeclared; ALWAYS surfaced "declared, unverified" (M-I4)

// --- attach a checked bound to a value (basis governs the tag — never upgraded) ---
fn attach<T>(x: T, b: Bound) -> Approx<T>            // tag = strength implied by b.basis (M-I2/M-I3/M-I4)

// --- composition: thread through the trusted kernels; strength by MEET; basis re-derived ---
fn compose_eps<T>(a: Approx<T>, b: Approx<T>, rule: ErrRule) -> Result<Approx<T>, NumErr>
    // wraps mycelium-numerics::compose_error_bound; Err(NoRule) when no sound ε propagation exists (REFUSE, never fabricate)
fn union_delta(ds: &[Approx<_>]) -> Result<Bound, NumErr>   // wraps ::prob union(δ..) = min(1, Σδ); meet strength
fn accuracy_to_probability(a: Approx<_>, tol: Real) -> Bound // the ONE legal cross-kernel rule (ADR-010 §4)
    // within tol ⇒ inherits the accuracy confidence; outside ⇒ honest worst case δ = 1 (never a silent tighten)

// --- inspect / EXPLAIN (C3) ---
fn explain(b: Bound) -> Explanation     // {kind, eps|delta, norm, basis, strength} + WHY (theorem | fit | declared-unverified)

// --- check a claimed certificate via the M-203 tier-i checker (re-validates) ---
fn check(claim: Certificate) -> Result<Valid, CheckErr>
    // Ok(Valid) when the claim is sound (looser-or-equal to the re-derivation);
    // Err(Rejected{recomputed, claimed}) when the claim is TIGHTER than re-derivable (never a silent pass);
    // Err(Malformed) when the certificate is ill-formed.

enum NumErr   { BadEps, BadDelta, NoRule, NormMismatch }   // structured refusal (RFC-0013 record)
enum CheckErr { Rejected{ recomputed: Real, claimed: Real }, Malformed }
```

> **Carrier resolution (homing math.md §7 Q1 / dense.md §7 Q5):** `Approx<T>` is the *thin* form — a value with its `Meta`-attached `{Bound, strength}` (RFC-0001 §4.3), **not** a distinct numeric type and **not** a kernel change. It is a *view* over the existing `Meta`, so it composes with content-addressing (ADR-003: `Meta` is not identity) and adds no new representation kind. `math`/`dense` "carry an `Approx<T>`" = "thread the `Meta` bound"; the ergonomics of *where* that surfaces at the call site is the cross-cutting tension A (§7 Q4), not a fork of the value model.

## 4. Guarantee matrix (the load-bearing deliverable — RFC-0016 §4.5)

Rows = exported helper ops. To be encoded as a checked table (the RFC-0003 §4 / `bound.schema.json` template) and asserted in tests once code lands — never prose only. **Tag legend:** the `Strength` column is the lattice tag of the *helper's own result* (`Exact ⊐ Proven ⊐ Empirical ⊐ Declared`). Most helpers are **structural** — they construct, attach, project, or check a bound — so the helper *itself* is `Exact` (it computes no approximate numeric quantity); the strength of the *carried bound* it produces is **whatever its `basis` supports, never set by this module** (that is the honesty crux, justified below).

| Op | Guarantee tag (of the helper op) | Fallibility (explicit error set) | Declared effects | EXPLAIN-able? |
|---|---|---|---|---|
| `exact` | `Exact` | total | `none` | n/a (no bound — M-I1) |
| `value_of` / `bound_of` / `strength_of` | `Exact` (projections) | total | `none` | yes (surfaces the carried `{bound, strength}`) |
| `error_bound` | `Exact` (constructs the carrier; ε is the caller's datum) | `Err(NumErr::BadEps)` when `eps < 0` | `none` | yes (the constructed bound + its basis) |
| `prob_bound` | `Exact` (constructs the carrier) | `Err(NumErr::BadDelta)` when `δ ∉ [0,1]` | `none` | yes (the constructed bound + its basis) |
| `declared` | `Declared` (basis = `UserDeclared`; **always** flagged) | total (the *assertion* never fails; it is just unverified) | `none` | yes (surfaced "declared, unverified" — M-I4/VR-5) |
| `attach` | `Exact` (structural); the **carried** tag = `basis`-implied (M-I2/M-I3/M-I4) | total | `none` | yes (which basis ⇒ which tag) |
| `compose_eps` | `Exact` (structural wrapper); **carried** strength = `meet` of inputs, basis re-derived | `Err(NumErr::NoRule)` when no sound ε rule exists (**REFUSE**) | `none` | yes (the composed certificate; *why* it refused) |
| `union_delta` | `Exact` (structural wrapper); **carried** strength = `meet` of inputs | `Err(NumErr::BadDelta)` on a malformed input δ | `none` | yes (the union certificate) |
| `accuracy_to_probability` | `Exact` (the *rule application* is exact); **carried** δ honestly = worst case when outside `tol` | total (outside `tol` ⇒ honest `δ = 1`, never a silent tighten) | `none` | yes (the ε→δ coupling record) |
| `explain` | `Exact` (pure projection of the certificate) | total | `none` | yes (it *is* the EXPLAIN artifact) |
| `check` | `Exact` (re-validation verdict; the trust anchor) | `Err(CheckErr::Rejected{recomputed, claimed})` / `Err(CheckErr::Malformed)` | `none` | yes (the verdict + the recomputed-vs-claimed delta) |

**Tag justification (VR-5 — downgrade rather than overclaim; this is the module's whole point):**

- **Why the helper ops are `Exact`, not `Proven`.** This is the crux that prevents a category error. The helpers *carry, compose, and check* bounds; they do **not** compute an approximate numeric result of their own, so the *helper* has no accuracy semantics and is `Exact` (the RFC-0016 §4.1-C2 "no accuracy semantics ⇒ `Exact`" rule, like `len`). The thing that *has* a strength tag is the **carried `Bound`** — and that strength is **whatever its `basis` supports**, set by the kernel/theorem, **never by this module**. The module's job is to keep that tag *honest in transit*, not to mint it. This is exactly why `std.numerics` is the right home for the discipline math/dense deferred: it owns the *carrier and the never-upgrade rule*, not the ε magnitudes or which transcendentals reach `Proven` (those are ADR-010 / the kernels' to supply — §7 Q2).
- **`declared` is `Declared` by construction** (basis `UserDeclared`, M-I4) and is **always** surfaced with the "declared, unverified" marker — the explicit, flagged escape hatch, never a silent assertion (VR-5).
- **Composition (`compose_eps` / `union_delta`) never upgrades.** Strength composes by **meet** (weakest-wins, ADR-010 §3): `Proven⊕Proven` stays `Proven` *only* under the cited affine-composition theorem; `Proven⊕Empirical → Empirical` (carrying the fewest trials); `…⊕Declared → Declared` — the basis is **re-derived to match** so M-I2…M-I4 hold (phase-2 §6.3). A composition with **no sound ε rule** is `Err(NoRule)` — an explicit refusal, never a fabricated bound (the M-204 `Refuse` discipline lifted to the library: `bit.*`/`trit.mul` had no rule and refused; this module refuses identically).
- **`check` is the re-validation anchor, and it can only *reject*, never silently upgrade.** It wraps the M-203 tier-i checker: a claim **looser-or-equal** to the kernel re-derivation is `Valid` (sound), a claim **tighter** than re-derivable is `Err(Rejected{recomputed, claimed})`, an ill-formed certificate is `Err(Malformed)`. Checker *incompleteness* surfaces as an explicit `Rejected`/`Malformed`, **never a silent pass** (RFC-0002 §2; phase-2 §6.3). Every certificate this module composes is re-checkable through `check` — the trust never rests on this Ring-1 layer.
- **`accuracy_to_probability` is the single sanctioned cross-kernel inference** (ADR-010 §4): within tolerance it inherits the accuracy confidence; **outside** tolerance it returns the honest worst case `δ = 1` — a downgrade, never a silent tighten. No other ε↔δ mixing is exposed.

## 5. §4.1 contract conformance (C1–C6)

- **C1 — never-silent (G2).** Every failure mode is an explicit `Result::Err` from `NumErr`/`CheckErr` that **propagates**: a negative ε is `Err(BadEps)`, an out-of-range δ is `Err(BadDelta)` (mirroring the M-203 range-checked constructor), a composition with no sound rule is `Err(NoRule)` (**refuse, never fabricate** — the M-204 posture), a too-tight claim is `Err(Rejected{recomputed, claimed})`. No helper returns a fabricated bound, a clamped ε, a default `δ`, or a silently-upgraded strength. The refusal carries a structured RFC-0013 diagnostic record (the violated invariant named, the recomputed-vs-claimed numbers attached), legible and re-propagable (I1), never swallowed.
- **C2 — honest per-op tag (VR-5).** *The* crux. The helper ops are `Exact` (they carry/compose/check, with no accuracy semantics of their own — §4); the **carried** bound's strength is *exactly* its `basis`-implied tag (M-I2/M-I3/M-I4) and is **never upgraded** — composition meets to the weakest input, `declared(...)` is always `Declared`+flagged, and the tier-i `check` can only confirm-or-reject, never strengthen. The module deliberately **homes the discipline, not the numbers**: which transcendentals legitimately reach `Proven`, and the concrete ε/`NormKind`, are ADR-010 / the kernels' to supply (§7 Q2) — this spec fixes the *carrier and the never-upgrade rule* and refuses to fabricate a magnitude.
- **C3 — no black boxes / EXPLAIN (SC-3/G11).** `explain` *is* a first-class EXPLAIN artifact: it projects `{kind, eps|delta, norm, basis, strength}` **and** *why* — which `ProvenThm{citation}`, which `EmpiricalFit{trials, method}`, or the `UserDeclared` "declared, unverified" marker. Every composed certificate is re-derivable via `check`; a refusal carries a diagnostic record naming the missing rule. No opaque heuristic sets a strength tag — the basis is always the inspectable reason.
- **C4 — content-addressed, value-semantic (ADR-003 / RFC-0001).** Helpers are pure functions of their inputs (no ambient rounding mode, no hidden state); `Approx<T>` values are immutable. The carried `Bound` lives in `Meta`, and **metadata is not identity** (ADR-003) — two values equal up to their bound are the same value; attaching/re-deriving a bound never changes content-addressed identity.
- **C5 — above the small kernel (KC-3).** This module **adds no trusted code.** The ε affine algebra (M-201), the δ union/apRHL algebra (M-202), the certificate record + the tier-i checker (M-203) are the *trusted base*; `std.numerics` only *calls* them and surfaces an ergonomic shape. The trust anchor is the kernel checker, not this Ring-1 layer — a composed certificate is only as trusted as `check` confirms. No `wild`/FFI is introduced here (the kernels are pure Rust over the value model).
- **C6 — declared, bounded effects (RFC-0014).** Every helper is **pure** — `effects: none` across the matrix. No IO, no clock, no ambient randomness, no global rounding-mode read. (A future fixed-budget iterative bound-tightening, if any kernel grows one, would declare an `alloc`/iteration budget — but no helper here needs it; FLAGGED §7 Q3 if math/dense force a budgeted op.)

## 6. Grounding

- **The two bound kernels + the shared certificate + meet-composition:** **ADR-010** (Accepted) — the `ErrorBound` (ε, affine arithmetic) and `ProbBound` (δ, union/apRHL) monoids meeting at one `{ε, δ, strength}` certificate; `strength` composes by **meet**; the **single** sanctioned cross-kernel rule (accuracy→probability, §4); VSA crosstalk content explicitly **not** supplied here (§5); the **certificate-checker-in-Rust** trusted base (Tier i default / Tier ii verified-binary).
- **`BoundBasis` is universal:** **ADR-011** (Accepted) — every `Bound` carries a required `basis`, so the `Proven | Empirical | Declared` tag is grounded per op (M-I2/M-I3/M-I4); an `Exact` value carries no bound (M-I1). The projection: **`docs/spec/schemas/bound.schema.json`** (`ErrorBound{eps, norm, basis}` / `ProbabilityBound{delta, basis}` / `CrosstalkBound` / `CapacityBound`; `BoundBasis ∈ {ProvenThm{citation}, EmpiricalFit{trials, method}, UserDeclared}`; `NormKind ∈ {L1, L2, Linf, Rel}`, an extensible registry).
- **The landed surface this consumes:** **`mycelium-numerics`** — **M-201** `::error` (ε affine; sound `add`/`sub`/`mul`/`neg`/`scale`, radius→`eps`), **M-202** `::prob` (δ union; `min(1, Σδ)`, apRHL `[SEQ]`), **M-203** `::cert` (`Certificate{eps,delta,strength}`, `recompute_error`, `check_error_claim`/`check_union_claim` → `Valid`/`Rejected`/`Malformed`, `accuracy_to_probability`, `compose_error_bound`), all **Done 2026-06-09** (`docs/planning/phase-2.md` §6.1–§6.3); **M-204** (the interpreter's honest composition — the `Refuse`/never-fabricate discipline this module lifts, §6.4).
- **The per-op contract + matrix obligation + ring placement:** **RFC-0016** §4.1 (C1–C6), §4.2 (Ring-1 capability surfaces are certificate/EXPLAIN consumers), §4.3 (the `numerics` row: "honest ε/δ-bounded arithmetic helpers over the two bound kernels + shared certificate", grounded ADR-010 / M-201/202/203), §4.5 (every module ships a checked guarantee matrix); `docs/spec/stdlib/README.md` §5 (the "Numeric ε bounds ownership" seam — dense/math route ε through *this* module; constants are this module's to home, not theirs to fabricate).
- **The honesty lattice + value model + house rules:** **RFC-0001** (the `Exact ⊐ Proven ⊐ Empirical ⊐ Declared` lattice §4.3/§4.7, `Value`/`Repr`/`Meta`, invariants M-I1…M-I4); **VR-5** (honest tags, downgrade not upgrade), **G2** (never-silent), **G5** (disclose which bounds are proven vs empirical), **G11** (dual projection), **SC-3** (EXPLAIN), **KC-3** (small kernel), **ADR-003** (metadata is not identity); **RFC-0013** (the structured refusal record); **RFC-0014** (declared bounded effects).

## 7. Open questions (FLAGGED — resolve before ratification)

- **(Q1) The `Approx<T>` carrier shape — RESOLVED here, ratification-pending.** This module **homes** the carrier math.md §7 Q1 and dense.md §7 Q5 deferred. The resolution: `Approx<T>` is a *thin view* — a plain value with its `Meta`-attached `{Bound, strength}` (RFC-0001 §4.3), **not** a distinct numeric type and **not** a kernel change (so it composes with content-addressing, ADR-003, and adds no representation kind). `math`/`dense` "carry an `Approx<T>`" = "thread the `Meta` bound; never drop it." — *Disposition: proposed-resolved; the maintainer ratifies the carrier at M-501. The residual call-site-ergonomics question is Q4, not the carrier itself.*
- **(Q2) Which transcendentals/ops legitimately reach `Proven`, and the concrete ε magnitudes / `NormKind`.** This module fixes the **discipline** (basis-implied tag, never upgraded; meet on composition; refuse without a rule; `check` re-validates) and the **carrier shape** — but it deliberately does **not** fabricate *which* ops have a checked-side-condition cited theorem (so legitimately tag `Proven`, e.g. the Higham backward-error summation/inner-product bound dense.md §7 Q1 cites) versus settle for `Empirical`, nor their concrete ε / norm. Those are **ADR-010 / the M-201/M-203 kernels'** to supply, with side-conditions *checked at the call* (ADR-011; `bound.schema.json`). — *Disposition: the magnitudes and `Proven`-reachability are filled per-op when the kernels expose them; this spec FLAGs, never invents (the README §5 seam). Ties to RFC-0016 §8-Q1.*
- **(Q3) An iterative-refinement effect budget.** The kernels today are pure (`effects: none`). If a future bound-tightening helper uses bounded iteration to shrink an ε, that iteration count is a declared budget (C6/RFC-0014). No helper here needs it. — *Disposition: FLAGGED; keep pure unless math/dense force a budgeted op.*
- **(Q4) Ergonomics of the bound-carrying surface (tension A).** How much of the `Approx<T>` / EXPLAIN / tag machinery is *implicit-but-inspectable* at the call site vs always-explicit is the recurrent ergonomics-vs-contract tension, instantiated for *every* consumer of this module (math, dense, vsa, swap). This module fixes the *carrier and discipline*; it does **not** settle the call-site surface. — *Disposition: FLAGGED; needs one per-ring design pass, not a per-module answer. Ties to RFC-0016 §8-Q3 and README §5 ("Ergonomics vs the contract").*
- **(Q5) Tier-i vs Tier-ii checker selection at the library boundary.** ADR-010 offers a pragmatic Tier-i Rust checker (default) and a high-assurance Tier-ii verified binary (FloVer-style) for the strongest `Proven` tags. Whether `check` exposes the tier as a reified, EXPLAIN-able choice (so a caller can demand Tier-ii for a `Proven` claim) or defaults silently to Tier-i is a design call — a silent default would itself be a mild C3 gap. — *Disposition: FLAGGED; propose `check` carries the tier in its verdict so the assurance level is inspectable, never implicit. Confirm at ratification with the kernel owners.*
